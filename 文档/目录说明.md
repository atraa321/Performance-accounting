## 📝 文档维护

### 添加新文档

1. 在 `文档/` 目录创建新的 Markdown 文件
2. 更新本文件（目录说明.md）添加链接
3. 在 [README.md](../README.md) 中添加引用（如需要）
4. 更新 [更新日志.md](更新日志.md)

### 更新现有文档

1. 修改对应的文档文件
2. 更新文档中的"更新时间"
3. 在 [更新日志.md](更新日志.md) 中记录变更

### 删除过时文档

1. 确认文档确实过时
2. 移动到 `文档/归档/` 目录（如需保留）
3. 或直接删除（如不需要）
4. 更新本文件和相关引用

## 🎯 文档规范

### 文件命名

- 使用中文名称（便于识别）
- 使用 `.md` 扩展名
- 避免特殊字符
- 示例：`功能说明.md`

### 文档结构

```markdown
# 文档标题

## 📋 概述
简要说明文档内容

## 🎯 主要内容
详细说明

## 📝 注意事项
重要提示

## 🎉 总结
总结要点

---
**更新时间**: YYYY-MM-DD
**版本**: vX.X
**状态**: ✅ 已完成
```

### 格式要求

- 使用 Markdown 格式
- 使用表情符号增强可读性
- 使用表格组织信息
- 使用代码块展示代码
- 使用列表展示要点
