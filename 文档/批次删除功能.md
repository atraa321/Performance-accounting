# 批次删除功能

## 📋 功能说明

在批次管理页面新增了**删除批次**功能，允许用户删除不需要的批次及其所有相关数据。

**更新时间**: 2026-01-21

---

## ✨ 新增功能

### 1. 后端删除接口 ✅

**文件**: `app/api/routes/runs.py`

**接口**: `DELETE /runs/{run_id}`

**功能**:
- 删除指定批次
- 删除所有相关数据（原始数据、计算结果、对账数据、质量检查）
- 防止删除已锁定的批次

**删除的数据包括**:
- ✅ 原始数据表
  - 院发绩效表 (RawHospitalPerfItem)
  - 绩效发放名单 (RawRoster)
  - 夜班统计 (RawNightShift)
  - 判读费 (RawReadingFee)
  - 医师工作量 (RawDoctorWorkload)
  - 护士工作量 (RawNurseWorkload)
  - 手动工作量分配 (RawManualDoctorWorkloadPay)
- ✅ 计算结果
  - 汇总数据 (FactPaySummary)
  - 明细数据 (FactPayDetail)
- ✅ 对账数据 (ReconcileItem)
- ✅ 质量检查 (QcIssue)
- ✅ 批次本身 (RunBatch)

### 2. 前端删除按钮 ✅

**文件**: `frontend/src/pages/RunList.tsx`

**位置**: 批次管理页面 → 操作列

**功能**:
- 显示删除按钮（仅未锁定批次）
- 二次确认对话框
- 删除成功提示
- 自动刷新列表

## 🎯 使用方法

### 删除批次

1. 进入**批次管理**页面
2. 找到要删除的批次
3. 点击操作列的**删除**按钮
4. 在确认对话框中点击**确定删除**
5. 系统自动删除并刷新列表

## 🔒 安全限制

### 1. 已锁定批次无法删除

**规则**: 状态为 `LOCKED` 的批次不显示删除按钮

**原因**: 
- 已锁定的批次通常是已确认的正式数据
- 防止误删重要数据

**如需删除已锁定批次**:
- 需要先在数据库中手动解锁
- 或联系管理员处理

### 2. 二次确认

**规则**: 删除前必须确认

**提示信息**:
- "确定要删除此批次吗？"
- "删除后将无法恢复，所有相关数据都会被删除"

**按钮**:
- 确定删除按钮为红色（危险操作）
- 可以点击取消放弃删除

## ⚠️ 注意事项

### 1. 数据无法恢复

- 删除操作是**永久性**的
- 删除后无法恢复数据
- 请谨慎操作

### 2. 删除前建议

建议在删除前：
- ✅ 确认批次确实不需要
- ✅ 导出重要数据（如有需要）
- ✅ 检查是否有其他批次依赖此数据

### 3. 批量删除

当前版本不支持批量删除，如需批量删除：
- 需要逐个删除
- 或联系管理员通过数据库操作

## 🔧 技术实现

### 后端代码

```python
@router.delete("/runs/{run_id}")
def delete_run(run_id: int, db=Depends(get_db)):
    """删除批次及其所有相关数据"""
    run = db.get(RunBatch, run_id)
    if not run:
        raise HTTPException(status_code=404, detail="Run not found")
    
    # 检查是否已锁定
    if run.status == "LOCKED":
        raise HTTPException(status_code=400, detail="Cannot delete locked run")
    
    # 删除所有相关数据...
    db.delete(run)
    db.commit()
    
    return {"status": "deleted", "run_id": run_id}
```

### 前端代码

```typescript
// API 调用
delete: (runId: number) => api.delete(`/runs/${runId}`)

// 删除处理
const handleDelete = async (runId: number) => {
  try {
    await runApi.delete(runId)
    message.success('批次删除成功')
    loadRuns()
  } catch (error: any) {
    if (error.response?.data?.detail) {
      message.error(error.response.data.detail)
    } else {
      message.error('删除批次失败')
    }
  }
}
```
