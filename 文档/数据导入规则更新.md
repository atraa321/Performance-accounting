## 🎯 新增过滤规则

在导入"院发绩效表"时，现在会自动过滤以下数据：

### 1. 排除金额为 0 的项目 ✅

**规则**: 
- 金额为 `0`、`0.00` 或 `None` 的项目将被跳过
- 不会被导入到数据库

**原因**:
- 金额为 0 的项目不参与绩效计算
- 减少无效数据，提高计算效率
- 避免在结果中显示无意义的 0 值项目

**示例**:
```
项目名          金额
门诊收入        1000.00  ✅ 导入
住院收入        0.00     ❌ 跳过（金额为0）
药品收入        500.00   ✅ 导入
```

### 2. 排除合计行 ✅

**规则**:
- 项目名为"合计"、"总计"、"小计"、"汇总"的行将被跳过
- 最后一行数据（通常是合计行）将被跳过

**原因**:
- 合计行是汇总数据，不是实际的收入项目
- 避免重复计算
- 防止合计金额被当作单独项目处理

**示例**:
```
项目名          金额
门诊收入        1000.00  ✅ 导入
住院收入        2000.00  ✅ 导入
药品收入        500.00   ✅ 导入
合计            3500.00  ❌ 跳过（合计行）
```

## 🔍 详细逻辑

### 过滤流程

```
读取 Excel 行
    ↓
检查项目名是否为空？
    ↓ 否
检查是否是合计行？（项目名 = "合计/总计/小计/汇总"）
    ↓ 否
检查是否是最后一行？
    ↓ 否
转换金额为数值
    ↓
检查金额是否为 0 或 None？
    ↓ 否
导入到数据库 ✅
```

### 代码实现

```python
# 跳过空行
if item_name is None:
    continue

# 检查是否是合计行（通过项目名判断）
item_name_str = str(item_name).strip()
if item_name_str in ("合计", "总计", "小计", "汇总"):
    continue

# 转换金额
amount_decimal = to_decimal(amount)

# 排除金额为0的项目
if amount_decimal is None or amount_decimal == 0:
    continue

# 导入数据
session.add(RawHospitalPerfItem(...))
```

## 📝 注意事项

### 1. 数据对账

由于过滤了部分数据，导入的记录数可能与 Excel 行数不一致：
- **这是正常的**，被过滤的是无效数据
- 导入统计会显示实际导入的有效记录数

### 2. 历史数据

- 此修改**不影响**已经导入的历史数据
- 只对新导入的数据生效
- 如需重新导入，请删除旧批次后重新导入

### 3. Excel 格式要求

确保 Excel 文件格式正确：
- 第一行必须是表头
- 必须包含"项目名"和"金额"列
- 金额列应为数值格式
